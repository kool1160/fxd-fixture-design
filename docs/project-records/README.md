# FXD Professional Project Records

This directory contains the repo-native FXD project-record history and the audited professional binder set.

## Status authority

These binders are retrospective records through Milestone 31. They do not own or change current product-milestone status. Current status comes from the authoritative milestone registry and its linked GitHub issue and pull-request evidence.

## Audited binder set

- `print/FXD_Engineering_Binder_Volume_1_Milestones_01-25_Final.pdf` - closed audited Volume 1.
- `print/FXD_Engineering_Binder_Volume_2_Milestones_26-31_Final.pdf` - closed audited Volume 2.
- `AUDIT_MILESTONES_01_31.md` - audit result, record policy, and milestone evidence summary.
- `BINDER_SHA256SUMS.txt` - integrity hashes for the published print PDFs.
- `BINDER_DEPENDENCIES.md` - exact generator dependency, license, and redistribution record.

The repository publication path is generated directly from repo-native Python and approved repository assets. No editable DOCX source is part of this branch.

## Superseded records

Earlier binder Markdown and print files remain part of repository history, but the audited Volume 1 and Volume 2 files above supersede them for professional printing and project-history use. They should not be deleted solely because they are superseded; repository history remains the controlling audit trail.

## Volume policy

- Volume 1 is closed at Milestone 25.
- Volume 2 is closed at Milestone 31.
- Work after Milestone 31 belongs to a later binder volume or addendum when governed evidence authorizes publication.
- A closed volume may be replaced only to correct a factual, evidence, accessibility, or formatting defect.

## Record policy

The Engineering Constitution, Product Direction, accepted decisions, authoritative milestone registry, linked GitHub issues and pull requests, commits, tests, workflow runs, and review discussions remain controlling. Binder documents are subordinate retrospective engineering records and do not replace repository evidence or set milestone state.

No project record constitutes fixture certification, weld-process approval, structural validation, manufacturability approval, supplier quotation, safety certification, or production release. Source customer CAD remains immutable, deterministic engineering remains authoritative, AI remains advisory, and qualified human engineering approval remains mandatory.
